What this feed publishes

Each target directory can contain a raw aria2-next binary, BUILDINFO, an .ipk package for OPKG-based OpenWrt, and an .apk package for APK-based OpenWrt.

OPKG feed metadata is generated on this site today. APK files are published as standalone packages, not as a signed APK repository index.

Repository setup for OPKG

1. Detect the architecture that OpenWrt uses for package feeds:

opkg print-architecture | awk 'NF==3 && $3~/^[0-9]+$/ {print $2}' | tail -1

2. Add the matching aria2-next feed to your custom feeds file:

echo "src/gz aria2-next-static https://ysway.github.io/openwrt-aria2-next/$(opkg print-architecture | awk 'NF==3 && $3~/^[0-9]+$/ {print $2}' | tail -1)" >> /etc/opkg/customfeeds.conf

3. Refresh package indexes and install:

opkg update
opkg install aria2-next-static

Standalone APK install

1. Detect the architecture that APK uses:

cat /etc/apk/arch

2. Open the matching architecture block in the Packages tab and copy the direct link to the published .apk file.

3. Download and install it locally:

wget -O /tmp/aria2-next-static.apk <download-url-from-packages-tab>
apk add --allow-untrusted /tmp/aria2-next-static.apk

The --allow-untrusted flag is intentional here because this site publishes the package file itself rather than a signed APK repository.

Enable the service

The package ships with /etc/init.d/aria2-next and /etc/config/aria2-next.

Its UCI layout follows the OpenWrt project's aria2 package, so existing OpenWrt-style settings and multi-instance config aria2 sections can be reused after copying them to /etc/config/aria2-next. 

uci set aria2-next.main.enabled='1'
uci set aria2-next.main.dir='/mnt/data/downloads'
uci commit aria2-next
service aria2-next start
service aria2-next enable

Default runtime behavior

  • RPC listens on port 6800.
  • Downloads go to /mnt/sda1/aria2-next by default.
  • Rendered configs, session state, and DHT state live under /var/etc/aria2-next.
  • BitTorrent Local Peer Discovery and DHT are enabled by default.

Pick the files that match your device architecture.

Use .ipk for OpenWrt 24.10 and older. Use .apk for OpenWrt 25.12 and newer.

The raw aria2-next binary and BUILDINFO file are published alongside the packages for manual deployment and verification.

Build profile

These packages are built against official OpenWrt SDK container images and link aria2-next statically with libcurl, libtorrent-rasterbar, OpenSSL, libssh2, Boost headers, spdlog headers, and zlib.

Compression policy

UPX compression is applied where it is known to be safe and useful. The skip list currently includes the larger and more sensitive targets such as mips64, riscv64, and loongarch64.

Verification

The build pipeline verifies the final binary type, checks for dynamic dependencies, and runs basic smoke tests. Each published directory includes BUILDINFO so you can inspect dependency versions and the final static-link status.
